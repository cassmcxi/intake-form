# Client Intake Form

A multi-step client intake form for web design projects — built to replace vague email briefs and surface the details that actually matter.

## Why I Built This

The idea came from a real client interaction. A client emailed me a website revision request and I noticed inconsistencies between the hex codes they listed and their actual brand identity. When I dug into it, it was clear their request had been shaped by a ChatGPT conversation — and important details got lost in translation. I built this form so clients fill out the important questions themselves, directly. No third party in the middle. Clearer communication on both ends from the start.

As someone fairly new to the dev space, this project challenged me to separate my own aesthetic preferences from what actually serves the client on the UI/UX side. It also pushed me to finish something end-to-end — which is harder than it sounds when you have a lot of ideas running at once.
